# 罕见遗传代谢病临床智能平台项目说明

## 一、项目定位

本项目定位为面向医生端的“罕见遗传代谢病临床智能平台”，以 PKU、MMA 等专病病例为起点，利用真实临床病历、检验指标、基因报告、治疗方案、随访记录和专家处置意见，建设一个可结构化入库、可复盘专家决策、可支持科研队列分析的临床 AI 应用原型。

平台的核心不是单纯聊天机器人，而是围绕罕见病临床诊疗过程建立四层能力：

1. 扫描病历自动入库
2. 病例时间轴和指标趋势结构化
3. 专病诊疗决策支持
4. 科研队列和真实世界数据分析

系统只作为医生端临床决策支持工具，不替代医生诊断、处方和最终医疗决策。

## 二、项目背景

罕见遗传代谢病患者数量少、病程长、指标复杂、跨科室资料多，临床诊疗高度依赖专家经验。对于 PKU、MMA 等疾病，医生需要同时判断病史、代谢指标、基因型、营养方案、用药依从性、并发症风险、发育评估和复诊计划。

传统病历系统往往只保存原始文档，难以形成连续、可检索、可分析的专病数据资产。大量专家经验也沉淀在门诊处置意见、随访记录和病例复盘中，难以规模化传承。

本项目希望把杨教授等罕见病专家的临床数据和处置逻辑转化为可查询、可复盘、可研究、可辅助决策的智能系统。

## 三、核心功能模块

### 1. 病例助手

病例助手用于医生快速理解单个患者的完整病情。系统自动整理病例摘要、异常指标、缺失检查、治疗与营养方案、病程时间轴和下一步建议草稿。

主要能力包括：

- 自动生成病例摘要
- 提醒异常指标和缺失检查
- 展示关键指标趋势
- 生成下一步建议草稿
- 支持医生端 AI 问答
- 所有建议必须由医生确认后使用

### 2. 相似病例

相似病例模块用于回答“以前有没有类似患者，专家当时怎么处理”。系统根据疾病、基因型、异常指标、治疗方案、风险信号和专家复盘要点计算相似度。

主要能力包括：

- 展示相似病例列表
- 标注相似度和相似原因
- 展示既往处理方式
- 汇总专家处置摘要
- 支持医生参考既往病例经验

### 3. 科研队列

科研队列模块用于真实世界研究和专病数据分析。医生或研究者可以按照疾病、基因型、年龄、指标、治疗方案等条件筛选患者，形成研究队列。

主要能力包括：

- 按疾病、基因型、年龄、指标、治疗方案筛选患者
- 展示队列统计
- 支持疗效分析
- 支持误诊路径分析
- 支持真实世界数据研究

### 4. 杨教授专家决策复盘助手

这是平台最具差异化的数据能力。系统不只是回答“这个病是什么”，而是复盘专家如何看病例、如何抓关键线索、如何补检查、如何调整治疗和复诊计划。

主要能力包括：

- 提取专家认为关键的症状和体征
- 标记必须补充或复核的检查
- 复盘指标升高时的饮食、营养和药物调整思路
- 提醒代谢危象、癫痫、发育问题、骨骼问题等风险
- 生成处置意见和复诊计划草稿

### 5. 诊后家属管理助手

家属端能力必须在医生端成熟后逐步开放，并且所有内容都要受医生控制。系统可把医生确认后的内容转化为通俗、可执行的诊后管理说明。

主要能力包括：

- 自动生成通俗版病情说明
- 生成饮食禁忌和推荐清单
- 生成用药提醒
- 生成复查提醒
- 提醒危险信号，如呕吐、嗜睡、抽搐、感染后代谢失控
- 家属上传新检验单后，系统先整理，再交给医生确认

## 四、数据处理流程

平台围绕临床病历形成闭环流程：

1. 上传或导入 PDF、扫描病历、检验单、基因报告、处方和随访记录
2. 系统识别文档类型并抽取结构化字段
3. 自动生成病例时间轴、指标趋势和证据页索引
4. 标记异常指标、缺失检查和待医生审核事项
5. 医生确认后进入专病数据库
6. 支持相似病例检索、临床问答、专家复盘和科研队列分析

## 五、AI 能力边界

平台中的 AI 能力用于资料整理和医生决策支持，重点强调证据、复核和医生确认。

AI 可以做：

- 整理病例摘要
- 汇总异常指标
- 提醒缺失检查
- 草拟下一步建议
- 检索相似病例
- 生成科研筛选和分析视图
- 把医生确认后的内容转化为家属可读版本

AI 不可以做：

- 独立给出最终诊断
- 独立开具处方
- 替代医生进行医疗决策
- 向家属直接输出未经医生确认的诊疗建议

## 六、项目价值

对医生：

- 减少阅读长病历和多份检查报告的时间
- 帮助快速抓住关键异常指标和风险信号
- 支持专家经验复盘和年轻医生培训
- 提高复杂罕见病随访管理效率

对科研：

- 把零散病历沉淀为结构化专病数据库
- 支持真实世界疗效分析
- 支持误诊路径和延误诊断研究
- 支持基因型、表型、治疗反应和长期结局关联分析

对患者管理：

- 提升复诊、复查、用药和饮食管理的连续性
- 降低家属理解复杂医嘱的门槛
- 在医生确认前提下提升诊后管理质量

## 七、当前原型范围

当前原型已经围绕 PKU 和 MMA 样本病例实现医生端工作台，包括病例助手、相似病例、科研队列、专家决策复盘、诊后家属管理草稿和 AI 问答接口。

原型支持本地病例数据兜底回答；配置 OpenAI API Key 后，可通过后端接口调用大模型生成基于病例上下文的医生端回答。

## 八、后续建设方向

下一阶段建议重点推进：

1. 增加真实 PDF 病历 OCR 和结构化抽取流程
2. 建立医生确认与数据质控工作台
3. 扩展 PKU、MMA、同型半胱氨酸血症等专病字段模型
4. 建立相似病例检索和专家处置知识库
5. 增加科研队列导出、统计分析和图表能力
6. 增加权限、脱敏、审计和伦理合规机制
7. 部署到可外部访问的云端环境，供团队评审和试用
